# ИСПРАВЛЕНИЯ В ПРИЛОЖЕНИИ CONTRACT FILLER

## Дата: 20.01.2026

## Исправленные проблемы:

### 1. Исправлен утильсбор - адрес компании вместо адреса клиента

**Проблема:** В шаблоне "Письма об утильсборе" в поле {{address}} заполнялся адрес клиента, 
а должен был заполняться юридический адрес компании.

**Исправление:** 
- В файле `app/main.py`, функция `util_create` (строка ~1532):
  - Изменено: `"address": address` → `"address": company_address`
  - Теперь в шаблон подставляется юридический адрес компании, а не адрес клиента

- В файле `app/templates/util_home.html`:
  - Добавлены пояснения к полям "Адрес клиента" и "Юридический адрес компании"
  - Пользователь теперь видит, какой адрес будет использован в шаблоне

### 2. Исправлены компании в ДКП - синхронизация с companies.json

**Проблема:** В ДКП использовался статический список компаний, который не синхронизировался 
с файлом companies.json (где хранятся настройки компаний).

**Исправление:**
- В файле `app/main.py`:
  - Удален статический список `DKP_COMPANIES` (строки 44-52)
  - Функция `dkp_home` (строка ~1569): теперь загружает компании из companies.json
  - Функция `dkp_create_form` (строка ~1796): теперь загружает компании из companies.json
  - Используется функция `load_companies()` для получения списка компаний

**Результат:** Теперь при добавлении новой компании в companies.json она автоматически 
появится в списке компаний для ДКП.

### 3. Унифицирован UI - все home-страницы теперь имеют единый стиль

**Проблема:** В некоторых HTML-шаблонах (util_home.html, kvit_home.html) использовалась 
одна колонка для выбора клиента, в других (invoice_home.html) - две колонки.

**Исправление:**
Все *_home.html файлы теперь имеют единую структуру:

- **Левая колонка (col-12 col-lg-6):** Выбор существующего клиента
- **Правая колонка (col-12 col-lg-6):** Создание нового клиента
- **Нижний блок:** Форма заполнения данных документа

**Обновленные файлы:**
- `app/templates/util_home.html` - унифицирован UI
- `app/templates/kvit_home.html` - унифицирован UI
- `app/templates/dkp_home.html` - создан заново с правильной структурой
- `app/templates/proxy_home.html` - создан заново с правильной структурой
- `app/templates/tpo_home.html` - создан заново с правильной структурой

**Примечание:** Файлы dkp_home.html, proxy_home.html, tpo_home.html имели размер 0 байт 
(не распаковались из архива), поэтому были созданы заново.

### 4. О сохранении данных year_car, color, probeg

**Анализ:** 
- Поля `vypusk`, `color`, `probeg` существуют в базе данных (таблица clients)
- В функции `dkp_generate` (строки 1901-1919) эти данные правильно сохраняются:
  - `vypusk` - год выпуска
  - `color` - цвет
  - `probeg` - пробег
- В функции `dkp_create_form` (строки 1742-1761) эти данные правильно подтягиваются из базы

**Вывод:** Механизм сохранения и подтягивания работает корректно. Если данные не сохраняются, 
возможные причины:
1. Данные не были введены при создании ДКП
2. База данных не обновляется (проблема с правами доступа)
3. Используется старая версия базы данных без этих полей

**Рекомендация:** Проверить, что:
- При создании ДКП все поля (год, цвет, пробег) заполнены
- Файл clients.db имеет права на запись
- При необходимости пересоздать базу данных (она автоматически создастся с правильной структурой)
