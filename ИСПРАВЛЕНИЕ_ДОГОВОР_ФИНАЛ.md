# ✅ ИСПРАВЛЕНО: Договор - CONTRACT_DATE и ORGAN

## 🔍 Найденные проблемы:

1. **Дата договора (CONTRACT_DATE)** - поле отсутствовало в форме
2. **Орган выдачи (ORGAN)** - данные не сохранялись и не подставлялись

## ✅ Что исправлено:

### 1. Добавлено поле "Дата договора" в форму

**Было:**
```python
MANDATORY_CONTRACT_FIELDS = ["CLIENT_FIO", "PASSPORT", "ORGAN", "VYDAN", "ADDRESS", "PHONE"]
```

**Стало:**
```python
MANDATORY_CONTRACT_FIELDS = [
    "CLIENT_FIO", "PASSPORT", "ORGAN", "VYDAN", 
    "ADDRESS", "PHONE", 
    "CONTRACT_NO", "CONTRACT_DATE"  # ← Добавлены!
]
```

**Результат:** Теперь в форме договора всегда есть поля:
- Номер договора
- **Дата договора** ← НОВОЕ!

### 2. Исправлена обработка ORGAN и CONTRACT_DATE

**ГЛАВНАЯ ПРОБЛЕМА:**
Функция `generate` обрабатывала только плейсхолдеры из шаблона, а ORGAN и CONTRACT_DATE не попадали в mapping!

**Было:**
```python
for key in t.get("placeholders", []):  # ← Только из шаблона!
    raw = (form.get(key) or "").strip()
    mapping[key] = raw
```

**Стало:**
```python
# Собираем ВСЕ плейсхолдеры, включая обязательные
placeholders = list(t.get("placeholders", []))
for mandatory in MANDATORY_CONTRACT_FIELDS:
    if mandatory not in placeholders:
        placeholders.append(mandatory)

# Теперь обрабатываем ВСЕ поля
for key in placeholders:  # ← Включая MANDATORY!
    raw = (form.get(key) or "").strip()
    mapping[key] = raw
```

**Результат:**
- ✅ ORGAN теперь попадает в mapping
- ✅ ORGAN сохраняется в базу
- ✅ ORGAN подставляется в документ
- ✅ CONTRACT_DATE работает так же

### 3. Договор сохраняется в профиль клиента

```python
# Сохраняем ссылку на созданный договор
if cid:
    save_generated_document(cid, "contract", out_id, filename)
```

**Результат:**
- ✅ Все договоры теперь видны в профиле клиента
- ✅ Можно отследить историю документов
