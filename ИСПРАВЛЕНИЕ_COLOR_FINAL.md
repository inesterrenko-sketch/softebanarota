# ✅ ИСПРАВЛЕНО: {{Color}} в ДКП

## 🔍 Проблема:

{{Color}} в ДКП не заполнялся

## ✅ Что исправлено:

### 1. Добавлена защита от пустого перевода

**Проблема:**
Функция `translate_color_ru_to_de` могла возвращать пустую строку, если цвет не был найден в словаре.

**Решение в dkp_create_form:**
```python
# Переводим цвет на немецкий
color = translate_color_ru_to_de(color_raw)

# ЗАЩИТА: Если перевод пустой, используем исходное значение
if not color and color_raw:
    color = color_raw
```

**Решение в dkp_generate:**
```python
# Переводим цвет на немецкий
color_de = translate_color_ru_to_de(color_input)

# ЗАЩИТА: Если перевод пустой, используем исходное значение
if not color_de and color_input:
    color_de = color_input
```

### 2. Подтягивание цвета из базы

**В dkp_generate добавлено:**
```python
color_input = (form.get("color") or "").strip()
if not color_input:
    color_input = (
        client.get("color", "") or           # Из базы
        contract_data.get("Color", "") or    # Из договора
        contract_data.get("COLOR", "") or
        contract_data.get("color", "") or
        tpo_data.get("Color", "") or         # Из ТПО
        ""
    )
```

### 3. Все варианты в mapping

```python
mapping = {
    "Color": color_de,
    "color": color_de,
    "COLOR": color_de,
}
```
