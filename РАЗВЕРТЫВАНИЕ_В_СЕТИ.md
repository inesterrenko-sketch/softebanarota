## 🏠 Локальное развертывание

### Windows (текущий вариант)
```bash
START.bat
```
или
```bash
START_SIMPLE.bat
```

### Linux/Mac
```bash
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
pip install -r requirements.txt
python3 -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Доступ: `http://localhost:8000`

## 🏢 Развертывание в локальной сети

### Вариант 1: Простой способ (без nginx)

**Windows:**
```bash
# В START.bat замените:
# --host 127.0.0.1
# на:
--host 0.0.0.0
```

**Linux/Mac:**
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

**Узнать IP адрес:**
- Windows: `ipconfig` → найти IPv4 адрес
- Linux/Mac: `ip addr` или `ifconfig`

**Доступ с других компьютеров:**
`http://192.168.1.XXX:8000` (замените XXX на ваш IP)

### Вариант 2: С nginx (рекомендуется)

#### Шаг 1: Установка nginx

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install nginx
```

**CentOS/RHEL:**
```bash
sudo yum install nginx
```

**Windows:**
Скачайте с https://nginx.org/en/download.html

#### Шаг 2: Настройка nginx

1. Скопируйте `nginx.conf` в `/etc/nginx/sites-available/contractfill`

2. Отредактируйте файл:
```bash
sudo nano /etc/nginx/sites-available/contractfill
```

3. Замените пути и IP адреса:
   - `/путь/к/вашему/contract_filler_app/` → реальный путь
   - `your-domain.com` → ваш IP адрес (например `192.168.1.100`)

4. Создайте симлинк:
```bash
sudo ln -s /etc/nginx/sites-available/contractfill /etc/nginx/sites-enabled/
```

5. Проверьте конфигурацию:
```bash
sudo nginx -t
```

6. Перезапустите nginx:
```bash
sudo systemctl restart nginx
```

#### Шаг 3: Запуск приложения как сервис

1. Скопируйте `contractfill.service`:
```bash
sudo cp contractfill.service /etc/systemd/system/
```

2. Отредактируйте пути в файле:
```bash
sudo nano /etc/systemd/system/contractfill.service
```

3. Запустите сервис:
```bash
sudo systemctl daemon-reload
sudo systemctl start contractfill
sudo systemctl enable contractfill  # Автозапуск
sudo systemctl status contractfill  # Проверка статуса
```

**Доступ:** `http://192.168.1.XXX` (без порта!)

## 🌍 Развертывание в интернете

### Шаг 1: Получите сервер

Варианты:
- VPS (DigitalOcean, Linode, Vultr) - от $5/месяц
- AWS EC2 (первый год бесплатно)
- Google Cloud Platform (бесплатные кредиты)
- Hetzner (дешевле всего)

### Шаг 2: Базовая настройка сервера

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка необходимых пакетов
sudo apt install -y python3 python3-pip python3-venv nginx git

# Создание пользователя для приложения
sudo useradd -m -s /bin/bash contractfill
sudo usermod -aG sudo contractfill
```

### Шаг 3: Установка приложения

```bash
# Переключитесь на пользователя
sudo su - contractfill

# Клонируйте или скопируйте файлы
mkdir ~/contractfill_app
cd ~/contractfill_app
# Скопируйте все файлы сюда

# Создайте виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### Шаг 4: Настройка nginx с SSL (Let's Encrypt)

```bash
# Установка Certbot
sudo apt install -y certbot python3-certbot-nginx

# Получение SSL сертификата
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# Certbot автоматически настроит nginx для HTTPS
```

### Шаг 5: Настройка firewall

```bash
sudo ufw allow 'Nginx Full'
sudo ufw allow OpenSSH
sudo ufw enable
```

### Шаг 6: Запуск как сервис

Следуйте инструкциям из раздела "Локальная сеть → Вариант 2"

**Доступ:** `https://your-domain.com`

## 🐳 Docker развертывание (самый простой!)

### Шаг 1: Установка Docker

**Ubuntu/Debian:**
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
```

**Windows/Mac:**
Скачайте Docker Desktop с https://www.docker.com/products/docker-desktop

### Шаг 2: Запуск

```bash
# Перейдите в директорию с docker-compose.yml
cd /путь/к/contractfill

# Запустите контейнеры
docker-compose up -d

# Проверьте статус
docker-compose ps

# Просмотр логов
docker-compose logs -f
```

### Шаг 3: Управление

```bash
# Остановка
docker-compose down

# Перезапуск
docker-compose restart

# Обновление после изменений
docker-compose up -d --build
```

**Доступ:** `http://localhost` или `http://ваш-ip-адрес`

## 🔒 Безопасность

### Обязательные меры:

1. **Смените дефолтные пароли** (если есть)

2. **Настройте firewall:**
```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp  # SSH
sudo ufw enable
```

3. **Регулярно обновляйте систему:**
```bash
sudo apt update && sudo apt upgrade -y
```

4. **Используйте SSL (HTTPS)** для защиты данных

5. **Настройте backup:**
```bash
# Автоматический backup базы данных
#!/bin/bash
tar -czf /backup/contractfill_$(date +%Y%m%d).tar.gz /путь/к/app/data/
```

## 🆘 Решение проблем

### Приложение не запускается

1. Проверьте логи:
```bash
sudo systemctl status contractfill
sudo journalctl -u contractfill -n 50
```

2. Проверьте порты:
```bash
sudo netstat -tulpn | grep :8000
```

3. Проверьте права доступа:
```bash
ls -la /путь/к/app/data/
```

### Nginx ошибки

1. Проверьте конфигурацию:
```bash
sudo nginx -t
```

2. Проверьте логи:
```bash
sudo tail -f /var/log/nginx/error.log
```

### Нет доступа из сети

1. Проверьте firewall:
```bash
sudo ufw status
```

2. Проверьте, что приложение слушает 0.0.0.0:
```bash
sudo netstat -tulpn | grep :8000
```

3. Проверьте router/firewall настройки (для доступа из интернета)
