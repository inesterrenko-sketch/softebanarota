# Изменения в приложении ContractFill

## 1. Исправление бага автораспознавания реквизитов

### Проблема
В функции `parse_requisites()` регулярные выражения для `bank_rs` (номер счёта) и `bank_ks` (корр. счёт) пересекались, что приводило к тому, что вместо корректного распознавания обоих счетов в документ вставлялся корр. счёт дважды.

### Решение
Обновлены регулярные выражения в файле `app/main.py` (строки ~389-391):

```python
# До исправления:
bank_rs = find(r"(?:Сч[её]тOKEN\r\s*сч[её]та)\s*(?:получателя)?\s*[:\-]?\s*(\d{20})")
bank_ks = find(r"(?:Корр\.?\s*сч[её]тOKEN\r\.?\s*сч[её]т|K\/C)\s*[:\-]?\s*(\d{20})")

# После исправления:
bank_rs = find(r"(?:Номер\s+сч[её]т[а-яА-Я]*|сч[её]т(?!.*Корр))\s*(?:получателя)?\s*[:\-]?\s*(\d{20})")
bank_ks = find(r"(?:Корр\.?\s*сч[её]т[а-яА-Я]*|Кор\.?\s*сч[её]т|K\/C)\s*[:\-]?\s*(\d{20})")
```

Используется negative lookahead `(?!.*Корр)` для предотвращения ложных срабатываний.

## 2. Новый документ: Доверенность

### Новые файлы

#### Шаблоны
- `app/templates/proxy_home.html` - страница выбора клиента для доверенности
- `app/templates/proxy_create.html` - форма создания доверенности
- `app/data/proxy_template.docx` - шаблон доверенности (скопирован из загруженного файла)

#### Данные компаний
- `app/data/companies.json` - база данных компаний для доверенностей

Компании по умолчанию:
1. **Автоберг** - Гожев Руслан Александрович, г. Ставрополь
2. **Автолюкс** - Башлыков Владимир Евгеньевич, г. Пятигорск  
3. **АТЦ Белгород** - Кибец Дмитрий Александрович, г. Белгород

#### Управление компаниями
- `app/templates/companies_admin.html` - админ-панель для добавления/удаления компаний

### Переменные шаблона доверенности
- `{{DATE}}` - дата выдачи доверенности (по умолчанию: 25 мая 2025 г.)
- `{{SROK}}` - срок действия (по умолчанию: 25 мая 2026 г.)
- `{{FIO}}` - ФИО поверенного
- `{{PASSPORT}}` - паспортные данные
- `{{ADDRESS}}` - адрес регистрации
- `{{CAR}}` - марка и модель авто
- `{{VIN}}` - VIN номер
- `{{COMPANY}}` - название компании доверителя
- `{{COMPANY_CITY}}` - город компании
- `{{COMPANY_ADDRESS}}` - адрес компании
- `{{DIRECTOR_FIO}}` - ФИО директора компании

### Новые маршруты
- `GET /proxy` - страница создания доверенности
- `POST /proxy/client` - выбор/создание клиента
- `GET /proxy/create` - форма заполнения
- `POST /proxy/generate` - генерация документа
- `GET /admin/companies` - управление компаниями
- `POST /admin/companies/add` - добавление компании
- `POST /admin/companies/delete/{id}` - удаление компании
- `POST /admin/companies/settings` - настройки по умолчанию

## 3. Профили клиентов

### Новые файлы
- `app/templates/client_profile.html` - страница профиля клиента

### Функции профиля

#### Просмотр данных клиента
- ФИО, паспорт, адрес, телефон
- Данные договора (номер, дата, авто, VIN)
- Финансы (таможня, ДКП, ТПО в BYN и EUR)

#### История
- **Реквизиты счетов** - история всех выставленных счетов с реквизитами получателей
- **Доверенности** - история созданных доверенностей
- **Договоры** - история договоров

#### Редактирование
- Возможность изменить все данные клиента
- Кнопка "Редактировать" открывает модальное окно

#### Быстрые действия
- Создать счёт на оплату
- Создать ТПО
- Создать доверенность
- Открыть договор

### Новые маршруты
- `GET /client/{id}` - профиль клиента
- `POST /client/{id}/update` - обновление данных

### API endpoints
- `GET /api/client/{id}/requisites` - последние реквизиты клиента
- `GET /api/client/{id}/last-proxy` - последняя доверенность клиента

## 4. База данных

### Новые таблицы

#### invoice_history
Хранит историю реквизитов для каждого выставленного счёта:
- id, client_id, created_at
- recipient_fio, bank_name, bank_inn, bank_kpp, bank_bik
- bank_ks, bank_rs, amount, purpose

#### proxy
Хранит историю созданных доверенностей:
- id, client_id, company_id, created_at
- proxy_date, srok, fio, passport, address
- car_model, vin

### Новые колонки в clients
- `last_proxy_fio` - последнее ФИО для доверенности
- `last_company_id` - последняя компания
- `tpo_price_eur` - сумма ТПО в евро
