# Исправления в приложении ContractFill

## Проблема 1: Доверенность подтягивала данные клиента вместо получателя платежа

### Что было неправильно:
При создании доверенности система автоматически подставляла паспортные данные и адрес **клиента**, хотя доверенность должна выписываться на **получателя платежа** (другого человека).

### Что исправлено:

**Файл: `app/main.py` (функция `proxy_create_form`)**

```python
# Было:
prefill = {
    "fio": requisites_fio or client.get("fio", ""),
    "passport": client.get("passport", ""),  # ❌ Брались данные клиента
    "address": client.get("address", ""),    # ❌ Брались данные клиента
    "car_model": client.get("car_model", ""),
    "vin": client.get("vin", ""),
}

# Стало:
prefill = {
    "fio": requisites_fio,  # ФИО поверенного (получателя платежа)
    "passport": "",         # ✅ Пустое поле - заполняется вручную
    "address": "",          # ✅ Пустое поле - заполняется вручную
    "car_model": client.get("car_model", ""),
    "vin": client.get("vin", ""),
}
```

**Файл: `app/templates/proxy_create.html`**

Добавлено предупреждение для пользователя:

```html
<div class="alert alert-warning py-2 small mb-2">
  <strong>Внимание:</strong> Доверенность создается на получателя платежа из реквизитов счета, а не на клиента.
  Паспортные данные и адрес заполните для поверенного (получателя платежа).
</div>
```

## Проблема 2: Ошибка при создании счёта на оплату

### Что было неправильно:
В функции `invoice_generate` переменная `cid` не была определена после проверки `client_id`, что вызывало ошибку при сохранении ссылки на сгенерированный документ.

### Что исправлено:

**Файл: `app/main.py` (функция `invoice_generate`)**

```python
# Было:
cid_raw = (form.get("client_id") or "").strip()
if not cid_raw.isdigit():
    raise HTTPException(status_code=400, detail="client_id is required")
client = db_get_client(int(cid_raw))  # ❌ cid не определен

# ... later in the code ...
if cid:  # ❌ NameError: name 'cid' is not defined
    save_generated_document(cid, "invoice", ...)

# Стало:
cid_raw = (form.get("client_id") or "").strip()
if not cid_raw.isdigit():
    raise HTTPException(status_code=400, detail="client_id is required")
cid = int(cid_raw)  # ✅ cid определен
client = db_get_client(cid)

# ... later in the code ...
save_generated_document(cid, "invoice", ...)  # ✅ Вызов без ошибки
```

## Проблема 3: Документы ДКП не отображались в профиле клиента

### Что было неправильно:
Таблица `generated_documents` могла не создаваться или не иметь правильную структуру в существующей базе данных, из-за чего ссылки на сгенерированные документы не сохранялись.

### Что исправлено:

**Файл: `app/main.py` (функция `db_migrate_extended`)**

Добавлена проверка и пересоздание таблицы при неправильной структуре:

```python
# Проверяем, существует ли таблица
cur.execute("SELECT name FROM sqlite_master WHERE type='table' AND name='generated_documents'")
if cur.fetchone() is None:
    # Таблица не существует, создаем её
    cur.execute("""
        CREATE TABLE generated_documents (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            client_id INTEGER NOT NULL,
            doc_type TEXT NOT NULL DEFAULT '',
            doc_id TEXT DEFAULT '',
            filename TEXT DEFAULT '',
            created_at TEXT DEFAULT CURRENT_TIMESTAMP,
            FOREIGN KEY (client_id) REFERENCES clients(id)
        )
    """)
    print("Создана таблица generated_documents")
else:
    # Проверяем структуру таблицы
    cur.execute("PRAGMA table_info(generated_documents)")
    cols = [r[1] for r in cur.fetchall()]
    # Если нет колонки doc_type, пересоздаем таблицу
    if 'doc_type' not in cols:
        cur.execute("DROP TABLE generated_documents")
        cur.execute("""
            CREATE TABLE generated_documents (...)
        """)
        print("Пересоздана таблица generated_documents")
```

## Как теперь работает приложение:

### Создание доверенности:
1. Вы выбираете клиента (для привязки к нему документа и получения данных об автомобиле)
2. Система автоматически подставляет ФИО **получателя платежа** из реквизитов счёта
3. Поля "Паспорт" и "Адрес" остаются **пустыми** — вы заполняете их вручную для поверенного
4. Данные об автомобиле (модель, VIN) берутся из данных клиента

### Создание ДКП:
1. Данные автоматически подтягиваются из:
   - Профиля клиента
   - Истории счетов (ФИО получателя платежа)
   - Последнего договора (`last_contract_json`)
   - ТПО (`last_tpo_json`)
2. После генерации данные сохраняются для будущего использования
3. Документ сохраняется в базе данных и отображается в профиле клиента

## Как запустить исправленное приложение:

```bash
cd contract_filler_app
pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Затем откройте http://localhost:8000 в браузере.

**Важно:** При первом запуске обновлённого приложения таблица `generated_documents` будет создана автоматически, и вы увидите сообщение в консоли.
